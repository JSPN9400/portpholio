Urban Air Quality & Pollution Trend Analysis (India)

Comprehensive analysis of pollution trends across major Indian cities to identify seasonal spikes and high-risk regions using 50,000+ environmental records.

SQL Python Power BI Data Analysis Window Functions

Problem Statement

Analyse pollution trends across major Indian cities to identify seasonal spikes and high-risk regions.

Dataset Overview

  • 50,000+ records across major cities
  • Columns: City, Date, AQI, PM2.5, PM10, Temperature, Humidity

Data Cleaning & Feature Engineering

  • Removed duplicates and inconsistencies
  • Handled missing AQI values using imputation
  • Created Month and Season columns for temporal analysis
  • Standardised pollutant metrics for cross-city comparison

SQL Analysis Techniques

  • Ranked cities by average AQI using window functions
  • Calculated yearly AQI growth trends
  • Compared seasonal pollution levels across regions
  • Applied PARTITION BY and ROW_NUMBER() for advanced ranking
Sample SQL Query
SELECT city, ROUND(AVG(aqi), 2) AS avg_aqi, ROW_NUMBER() OVER (ORDER BY AVG(aqi) DESC) AS rank FROM pollution_data GROUP BY city ORDER BY avg_aqi DESC;

Dashboard Overview

  • KPI cards displaying Average AQI and Highest Polluted City
  • Monthly trend line chart for pollution progression
  • City comparison visuals and regional analysis
  • Seasonal comparison breakdown by pollutant type

Key Insights

  • Winter months show significant AQI spikes in northern cities
  • PM2.5 contributes the highest pollution levels year-round
  • Certain industrial cities consistently rank high in annual averages
  • Correlation identified between temperature and pollution levels

Business Recommendation

Strengthen seasonal pollution monitoring systems and implement targeted emission control strategies during high-risk periods.

E-Commerce Sales & Customer Retention Analysis

Revenue and customer retention pattern analysis across 100,000+ transactions to identify growth opportunities and reduce churn risk.

SQL Python Power BI Data Analysis Subqueries

Business Problem

Analyse revenue trends and customer retention patterns to improve profitability and identify high-value customer segments.

Dataset Information

  • 100,000+ transaction records spanning 24 months
  • Columns: Order_ID, Customer_ID, Order_Date, Revenue, Cost, Region

KPIs Calculated

  • Total Revenue and Period-over-Period Growth
  • Profit Margin by region and customer segment
  • Month-over-Month (MoM) Growth Percentage
  • Average Order Value (AOV) trends
  • Customer Retention Rate by cohort

SQL Techniques Used

  • Complex JOINs for customer and order data consolidation
  • Aggregations with GROUP BY and HAVING clauses
  • Subqueries for cohort analysis and retention calculation
  • Window Functions (ROW_NUMBER, RANK, LAG) for trend analysis
  • LAG() function for month-over-month growth calculation
Sample SQL Query
SELECT DATE_TRUNC('month', order_date) AS month, SUM(revenue - cost) AS profit, LAG(SUM(revenue - cost)) OVER (ORDER BY DATE_TRUNC('month', order_date)) AS prev_profit, ROUND(((SUM(revenue - cost) - LAG(SUM(revenue - cost)) OVER (ORDER BY DATE_TRUNC('month', order_date))) / LAG(SUM(revenue - cost)) OVER (ORDER BY DATE_TRUNC('month', order_date)) * 100), 2) AS mom_growth FROM orders GROUP BY DATE_TRUNC('month', order_date) ORDER BY month;

Dashboard Overview

  • Revenue trend chart showing monthly progression
  • Customer segmentation analysis by AOV and purchase frequency
  • Regional performance breakdown with profit margins
  • KPI summary cards for executive overview
  • Retention cohort heatmap

Key Insights

  • Revenue growth varies significantly by region (15-40% range)
  • Top 20% of customers contribute 65% of total revenue
  • Specific customer segments show early churn risk patterns
  • Seasonal trends influence regional performance differently

Business Recommendation

Implement targeted retention campaigns for high-value segments and region-based marketing strategies to capitalize on growth opportunities.
